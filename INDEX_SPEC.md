# index.html（3Dダンジョン）仕様書

> 3D版の仕様・操作設計・アセット定義をまとめるドキュメント。
> `playtest.html` の2Dゲームデザインを踏襲し、一人称視点3Dで再構築する。
> 実装手順は [ROADMAP.md](ROADMAP.md) を参照。

## 位置づけ

```
prototype.html  →  playtest.html  →  index.html（本仕様書）
（バランス調整）    （デザイン確認）    （3D移植・完成）
```

`playtest.html` のゲームロジック（種族AI・クリスタル・バトル・ショップ・勝敗）は**ほぼそのまま流用**し、以下を3D版として独自に作り込む：

- 一人称レイキャスティング3Dビュー
- グリッドベース一人称移動（前進・後退・左右回転・左右ストレーフ）
- 4方向スプライト表示（前後左右）
- ミニマップ・コンパス・全体マップ（Mキー）
- UIレイアウト（3Dビュー＋ミニマップ＋Message Log＋Status＋Factions）

## 世界の構造

### 迷路

- グリッドサイズ：**51 × 51 固定**
- 生成アルゴリズム：DFS再帰バックトラッカー（`generateMaze`）
- ループ追加：200カ所程度（`addLoops` で回り道を増やす）
- セル種別：0=通路、1=壁
- ブロック内に通路セルがゼロの場合は迷路を再生成する（`generateMazeUntilValid`）

### 5×5ブロック分割

マップ全体をプレイ範囲（row/col 1〜49）で **5×5 = 25ブロック** に分割し、各ブロックに**クリスタル1個**を配置（計25個）。

**ブロック列幅 / 行高：10, 10, 9, 10, 10**（歪みを中央に集約）

| ブロック列index | col 範囲 |
|---|---|
| 0 | 1〜10 |
| 1 | 11〜20 |
| 2 | 21〜29 |
| 3 | 30〜39 |
| 4 | 40〜49 |

（行も同一）

### 4陣営の初期陣地（L字3ブロック）

| 陣営 | 初期ブロック（行index, 列index） | 形状 |
|---|---|---|
| 人間族（左上） | (0,0), (0,1), (1,0) | L字 |
| ゴブリン（右上） | (0,4), (0,3), (1,4) | 逆L字 |
| リザード（左下） | (4,0), (4,1), (3,0) | L字 |
| オーガ（右下） | (4,4), (4,3), (3,4) | 逆L字 |

残り**13ブロック**は中立スタート。

### クリスタル連結ルール

- 各ブロックを頂点、上下左右4方向隣接ブロックをエッジとするグラフ上で BFS を実行
- 自陣営の初期3ブロックのいずれかから連結して到達できるクリスタルを **「有効（valid）」** とする
- クリスタル所有者が変わるたびに全陣営の連結判定を再計算
- **有効クリスタルのみ** 以下の機能を利用可能：
  - スポーン（AIユニット生成）
  - 回復（`healAtCrystal`）
  - ショップ（`openShop`）
- 飛び地クリスタルの見た目は通常の陣営色のまま（グレーアウトなし）
- プレイヤーが飛び地クリスタル上で Q/E を押してもターン消費せずログ警告のみ
- ユニット上限の計算は**所有クリスタル全体**（連結問わず）の合計で行う

ブロック占有陣営の色は**3Dビューの床**に α=0.15 で乗算され、占領に連動してリアルタイムに変化する（##12 改修以降）。Mキー全体マップ側にもブロック単位の陣営色オーバーレイが残るが、ミニマップ（近傍7×7）は地形のみを表示する。

## 3Dビューの環境演出（##12 / ##17 改修）

**目的**: プレイヤーをミニマップ依存から脱却させ、3Dビュー内で戦況を体感できるようにする（##12）。さらに、Wolfenstein 風の単色描画から脱却し、リアル系ダンジョンの質感に寄せる（##17）。

### クリスタルの光（光の柱）

- 各クリスタルの位置を中心に、**床と壁を陣営色のグロー**で照らす
- 床: 半径3セル / 中心 α 0.65 / 線形フォールオフ / 加算合成
- 壁: 床の70%濃度（白飛び抑制）。レイヒット点のワールド座標から距離算出
- 中立クリスタルもグレーでうっすら光る（占領可能の存在を伝える）
- **角を曲がった先のクリスタル**でも、見える床ピクセルに光が漏れる演出が成立する（occlusionは壁描画レイヤーで自然に処理される）

### ブロック陣営占有色（床ティント）

- 各床ピクセルが属するブロックのクリスタル所有者に応じて、**陣営色を α=0.15 で床に乗せる**
- 中立ブロックは塗らない（無彩色のまま、未占領感を残す）
- プレイヤーが歩いている空間自体で「いまどの勢力圏にいるか」が分かる

### ミニマップの役割再設計

- **3視点の役割分担**:
  - 3Dビュー: 探索・索敵・発見・遭遇
  - ミニマップ（近傍7×7）: 迷子防止と方角のみ
  - Mキー全体マップ: 戦略・勢力図
- ミニマップでは**プレイヤー中心の円（チェビシェフ距離 R=2、5×5相当）内のみ**にクリスタル/NPCを描画。境界1セル分は α フェードで自然に
- 円の輪郭は薄い点線で示し「ここから外は見えない」を伝える

### クリスタルの発見ステート

- 各クリスタルは `discovered: boolean` フィールドを持つ
- **発見条件**: 一度でも人間族（プレイヤー or 人間族NPC）の所有になったクリスタル
  - 初期人間族クリスタル（陣営本拠の3個）はゲーム開始時から発見済み
  - 占領で `owner` が `'human'` に変わるとき `discovered = true` をセット
  - 一度発見されたら**永続**（敵に再奪取されても発見状態は維持）
- **可視性**:
  - ミニマップの三角マーカー: 発見済みクリスタルのみ表示（円内フィルタとAND）
  - Mキー全体マップの三角マーカー: 発見済みクリスタルのみ表示
  - Mキー全体マップのブロック領域色オーバーレイ: 発見状態に関わらず常時表示（戦況の概略は確認できる）
- **3D ビューの床/壁グロー**: 発見状態に関わらず常時表示（探索体験を損なわないため）

### テクスチャマッピング（##17 追加）

3枚の石造り PNG テクスチャ（512×512、シームレスタイル）を `assets/textures/` に配置。

| ファイル | タイル方向 | 用途 |
|---|---|---|
| `wall_stone.png` | 横シームレス | 壁（1セル＝CELL_SIZE=40 ワールド単位＝テクスチャ幅） |
| `floor_stone.png` | 4方向シームレス | 床 |
| `ceiling_stone.png` | 4方向シームレス | 天井 |

描画方式：

- **壁**：`drawImage(img, texX, 0, 1, texH, x0, y0, colW+0.5, wallH)` で 1px 列スライスを縦ストレッチ。perspective-correct
- **床/天井**：per-pixel ImageData サンプリング。各ピクセルからワールド座標逆算 → `(wx * texPerCell) & mask` で UV ルックアップ
- 起動時にオフスクリーンキャンバス経由で生 `Uint8ClampedArray` をキャッシュ（`_floorTexData` / `_ceilTexData`）。テクスチャ未ロード時は旧来の単色描画にフォールバック

### 距離ライティング（##17 追加）

`render3d.js` 上部に定数ブロックを定義：

```js
const LIGHT_SCALE_FLOOR   = 160;   // 床/壁の exp フォグスケール
const LIGHT_SCALE_CEILING = 110;   // 天井のフォグスケール（より急峻）
const WALL_DIR_SHADE_V    = 0.60;  // 縦壁(E/W面)を 60% に減光
const FOG_R = 14, FOG_G = 9, FOG_B = 5;  // 環境光カラー（暖茶の闇）
```

4要素の組み合わせ：

| 要素 | 内容 |
|---|---|
| ① 指数フォールオフ | `shade = exp(-d / SCALE)` で線形フォグから置換。奥が指数的に深い闇に沈む |
| ② 方向別シェーディング | 壁が縦(E/W面=`wall.a.x === wall.b.x`)なら shade に `WALL_DIR_SHADE_V` を乗じる。廊下の角に立体感 |
| ③ 環境光カラー | 黒オーバーレイではなく `(14,9,5)` の暗い暖茶へ減衰。`final = tex * shade + FOG * (1-shade)`（per-pixel）／ 壁は `fillStyle = FOG_RGB` の半透明オーバーレイ |
| ④ 天井スケール分離 | 床=160、天井=110 で**天井がより速く闇に沈む**（上方向の閉塞感） |

クリスタルグローは床/天井で**共通計算1回**で両方に適用（同じ `(wx, wy)` を使うため）。天井は `CRYSTAL_GLOW_WALL_RATIO = 0.7` で減衰。

### スプライト環境光（##17 追加）

スプライト位置のワールド座標で「距離フォグ + クリスタルグロー」を計算し、その明るさで描画。

| シチュエーション | 結果 |
|---|---|
| 遠景・光源なし | 暗いシルエット（不透明） |
| 近景・光源なし | フル輝度 |
| 遠景・光源近く | 光源色で照らされる |
| 近景・光源近く | やや色着き |

実装：

- 使い回しオフスクリーンキャンバス（`_spriteTintCanvas`）にスプライトを描画 → `source-atop` で色オーバーレイ
- 順序：① 黒 `rgba(0,0,0, 1-distShade)` で減光 → ② グロー色 `rgba(R,G,B, tintA)` で着色（`tintA` は `SPRITE_GLOW_TINT_MAX = 0.45` 上限）
- オフスクリーン経由により `source-atop` がスプライト形状のみに作用（バウンディングボックスへの誤適用を回避）
- `globalAlpha` 減光は使わない（半透明化を避け、不透明シルエットを維持）
- クリスタル自身は光源なので**減光のみ・グロー着色なし**
- どちらの効果も不要な場合は元の `img` を直接 `drawImage`（高速パス）

### 描画パイプライン

床/天井は `render3d.js` の `drawFloor()` でピクセル単位描画（ImageData ベース）。床と天井は対称（horizon 距離が同じピクセルは同じ corrDist）なので**1ループで両方の ImageData を埋める**。ピクセルごとに:

1. 画面 Y → corrDist → ワールド座標 `(wx, wy)` 逆算
2. 距離フォグ計算（床=`exp(-d/160)`、天井=`exp(-d/110)`）
3. テクスチャ UV サンプリング → `tex_color * shade + FOG * (1-shade)`
4. ブロック陣営色を加算（`cellBlockIdx` 平坦インデックス + `_blockTint*` Float32Array、床のみ）
5. 視錐台内クリスタルからのグロー寄与を共通計算 → 床はフル、天井は 0.7 倍で加算
6. 上限 255 でクランプ

壁は `drawView3D()` の rays ループでカラム描画：

1. corrDist → wallH 計算、`shade = exp(-corrDist/160) * dirShade`
2. 1px 列スライスを `drawImage` で縦ストレッチ
3. 距離フォグ：`fillStyle = FOG_RGB`, `globalAlpha = 1-shade` でカラム全体に重ねる
4. クリスタルグローがあれば `globalCompositeOperation = 'lighter'` で加算オーバーレイ

事前計算: `_rayCos` / `_raySin` / `_cosCorr` を per-column キャッシュ、`_glowsBuf` / `_nGlows` を FOV コーン+αマージンで絞り込み（壁/スプライトループからも再利用）。

性能：`drawView3D` ≈ 14ms/frame（約 70fps 相当）。per-pixel 床/天井ループが支配的。

### NPCアグロ

全種族のユニットが `aggroRange = 2`（マンハッタン距離）以内の異陣営NPCまたはプレイヤーを検知すると追跡を開始する。両方が射程内にいる場合は**距離が近い方を優先**し、同距離ならプレイヤーを優先する。

- **対プレイヤー**：goblin / lizard / ogre のみ発動。隣接1マスに到達した時点でバンプ攻撃 → ターン制バトル起動
- **対NPC**：全種族（人間族含む）で発動。隣接1マスから敵NPCのマスへ踏み込み、次ターンの同マス戦闘で解決
- **無効化条件**：retreating / healing 中はアグロ判定をスキップ

> 経緯：##5 でプレイヤーアグロ実装、##19 で人間族追加と NPC 同士のアグロを導入。種族個性として `aggroRange` を差別化していた時期もあったが、計測の結果（##19-2）全種族 2 に統一して安定。

## プレイヤー

### 初期ステータス

| 略称 | 名称 | 初期値 |
|---|---|---|
| HP | Hit Points | 45 |
| ATK | Attack | 7 |
| REC | Recovery | 5（自陣クリスタル上の1ターン回復量） |
| AGI | Agility | 10（逃走成功率に影響） |
| GOLD | 所持金 | 40（##19-8 で 0→40 に変更：序盤の装備購入を可能に） |

### 装備スロット

- 武器 `weapon` × 1
- 防具 `armor` × 1
- アクセサリ `accessory` × 1

新規購入時、既存装備は**半値で自動買取**。装備プールは playtest.html と同一（14種）。

### 死亡時処理

- 人間クリスタルが1つだけ残っている状態で死亡 → そのクリスタルが敵に陥落 → **ゲームオーバー**
- 2つ以上残っている場合：
  - 有効な自陣クリスタルにリスポーン（有効クリスタルが無い場合は全所有クリスタルからフォールバック）
  - **GOLD 半減**
  - **装備1個ロスト**：`accessory → armor → weapon` の優先順で装備中の先頭1個を失う（##19-8 で順序固定化、武器が残りやすい設計）
  - HPは最大値まで回復

### ショップ仕様（##19-8 改訂）

- 自陣の有効クリスタル上で `E` キーを押すと開店
- **開店ごとに 1 ターン消費**（旧仕様の無限リロード抑制）

- `Esc` で閉じる場合はターン消費なし

**画像仕様**

| 項目 | 値 |
|---|---|
| サイズ | 512 × 512 px（パワーオブツー） |
| フォーマット | PNG（αチャンネル不要） |
| カラースペース | sRGB |

### 状態管理規約（Game 名前空間）

実行時に変わる値はすべて `Game.state.*` / `Game.flags.*` 経由で参照する（[js/state.js](js/state.js) で定義）。純粋な定数（`CANVAS_W` / `FACTIONS` / `UNIT_DEFS` 等）は `config.js` に置く。

```js
Game.state = {
  // World
  grid, walls, wallsH, wallsV, explored,
  // Entities
  monsters, crystals, crystalAtCell, crystalByBlock, humanAutoSpawnIndex,
  // Player
  player,
  // Turn / Log
  worldTurn, messageLog,
  // Modal / interaction
  battleState, shopItems, shopSelectedIdx, onCrystal,
};
Game.flags = {
  monstersAnimating, pendingBumpCheck, battleNeedsRerender,
  gameEnded, fullMapOpen, restartConfirmOpen,
};
```

主要な型は `state.js` の JSDoc `@typedef` で定義（`Crystal` / `Monster` / `Player` / `BattleState` / `LogEntry` / `Wall` / `EquipItem`）。

**lookup テーブル（O(1) 参照）**
- `Game.state.crystalAtCell[r][c]` … (r,c) 位置のクリスタル
- `Game.state.crystalByBlock[bR][bC]` … (bR,bC) ブロックのクリスタル
- `cellToBlock[r][c]` … (r,c) セル → 所属ブロック `[bR,bC]`（config.js の定数）
- `cellBlockIdx[r * GRID_SIZE + c]` … セル → ブロック平坦インデックス（0〜24、ブロック外は sentinel = 25）。床のホットループ用に条件分岐ゼロで陣営色を引ける（##12 改修）

**描画最適化**
- 壁は `Game.state.wallsH` / `wallsV` に水平・垂直で分割保存。`castRays` はレイ方向の符号で半分を即カット（実測 2.09x）
- `_hitsBuffer`（render3d.js）を固定長 RAY_COUNT で再利用、毎フレーム配列を新規作成しない
- `_spritesBuffer` も同様に再利用
- `_floorImageData`（render3d.js）を初回作成して使い回し、`putImageData` ベースで床ピクセル描画
- `_rayCos` / `_raySin` / `_cosCorr` を per-column キャッシュ（`Float32Array(R.w)`）して内側ループの三角関数呼び出しゼロに
- `_glowsBuf` / `_nGlows` で FOV コーン+αマージンに収まるクリスタルだけに絞り込み、床ループと壁ループから共有

ブラウザコンソールで状態の即時確認が可能：

```js
Game.state                                      // 全状態スナップショット
Game.state.crystals.map(c => [c.r, c.c, c.owner])
JSON.stringify(Game.state, null, 2)             // セーブデータ相当
```
