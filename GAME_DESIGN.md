# ゲームデザインドキュメント

## タイトル（仮）：Dungeon Liberate

> 3Dダンジョン探索 × リアルタイム陣地管理
> モンスターの生態系を読み、ダンジョンを人間族の手に取り戻せ。

---

## コンセプト

1つの巨大ダンジョンに複数のモンスター種族が陣地を持ち、互いに縄張り争いをしている。
プレイヤーは人間族のハンターとして潜入し、モンスターの**スポーナーを封印**することでダンジョンを解放していく。
すべてのスポーナーを封印することが最終ゴール。
人間族の拠点がモンスターに占拠されるとゲームオーバー。

---

## 世界観

- 人間族・ゴブリン族・魔族など複数の種族が存在する世界
- ダンジョンは各モンスター種族の陣地が入り組んでいる
- モンスター同士も常に陣地の取り合いをしており、生態系が動的に変化し続ける
- プレイヤーはその生態系を観察・利用しながらダンジョン攻略を進める

---

## ゲームゴールとゲームオーバー

| 条件 | 内容 |
|---|---|
| **クリア** | ダンジョン内の全スポーナーを封印する |
| **ゲームオーバー** | 人間族の拠点がモンスターに占拠される |

---

## 核心メカニクス：スポーナーシステム

### スポーナーとは
- 各モンスター陣地の中心に1つ存在する構造物
- 定期的にそのスポーナーの種族のモンスターをスポーンし続ける
- スポーナーの種族が変わると、スポーンされるモンスターも変わる

### 封印
- スポーナーには必ず**守衛モンスターが1体**待機している
- その守衛を倒すとスポーナーを封印できる
- 封印するとモンスターのスポーンが停止する

### 奪還（再解放）
- 封印されたスポーナーに別種族のモンスターが接触すると、その種族のスポーナーに変換される
- 封印は永続ではなく、守備が必要

### スポーナー変換の連鎖
```
[ゴブリン族スポーナー] → プレイヤーが封印
        ↓
[封印済みスポーナー] ← 魔族モンスターが接触
        ↓
[魔族スポーナー] ← 魔族モンスターがスポーンし始める
```

---

## モンスター生態系

### 陣地とエリア
- 各スポーナーを中心に **5×5マスのエリア**を陣地とする
- 陣地内のモンスター配置：
  - スポーナー：守衛1体が常駐
  - エリア内：徘徊モンスター（通常4体未満で安定）

### 溢れ出しメカニクス（閾値システム）
```
エリア内のモンスター数 ≥ 5（守衛1体 + 徘徊4体以上）
        ↓
閾値超過 → エリア外へ溢れ出す
        ↓
  ┌─────────────────────────────┐
  │ 近くに封印済みスポーナーあり → 奪還へ向かう  │
  │ なし → 人間族拠点へ向かう               │
  └─────────────────────────────┘
```

### モンスター同士の戦闘
- 異種族モンスターが接触すると**自動的に戦闘が発生**
- プレイヤーは戦闘画面を見なくてもよい
- ダンジョン全体図にモンスター数が常時表示される
- 戦闘発生と結果はメッセージログで確認できる

```
例：「ゴブリン族 vs 魔族が戦闘！ → ゴブリン族が勝利 / 魔族が1体減少」
```

### 生態系を利用した攻略
- モンスター同士を戦わせて数を減らし、閾値を下げる
- 弱ったところで漁夫の利を取りスポーナーを封印する
- 陣地争いの流れを読み、封印の優先順位を判断する

---

## プレイヤーの行動

### ダンジョン探索（3Dビュー）
- レイキャスティング方式のファーストパーソン視点
- グリッドベースの1マス単位移動（ターン制）
- 探索済みエリアはダンジョン全体図に反映される

### 戦闘（Wizardry風コマンドバトル）
- モンスターと接触するとバトル画面へ遷移
- コマンド選択式のターン制バトル
- 守衛を倒すことでスポーナーの封印が可能になる

### Gold収集
- 戦闘・探索によりGoldを獲得
- GoldはNPCハンターの増員・装備強化に使用

---

## NPCハンターシステム

### 概要
- プレイヤーがGoldを消費してNPCハンターを雇用・派遣できる
- NPCは自律的に行動し、プレイヤーの指示で役割を切り替えられる

### 指示できる役割

| 役割 | 条件 | 行動内容 |
|---|---|---|
| **守備** | 封印済みスポーナーが存在する | 指定スポーナーを守り、奪還を阻止する |
| **探索** | 常時可能 | 未踏エリアへ向かい、マップ開拓・Gold回収を行う |

### 運用戦略例
```
NPC_A → [守備] スポーナー(3,7)を守る
NPC_B → [探索] 未踏エリアへ向かう
NPC_C → [守備] スポーナー(9,2)を守る
```

---

## 成長・強化システム

### 装備・武器ビルド構築（主要やりこみ軸）
- モンスターを倒すと素材やGoldを獲得
- 装備・武器を強化・組み合わせてビルドを構築する
- ビルドの最適化が攻略効率に直結する

### NPC増員
- Goldを消費してNPCハンターを追加雇用
- NPCが増えることで、複数スポーナーを同時防衛できるようになる

## UI構成

### メイン画面（2分割）

```
┌──────────────────────┬──────────────┐
│                      │  ダンジョン   │
│   3Dダンジョンビュー   │   全体図     │
│   （プレイヤー操作）   │  陣地状況    │
│                      │  NPC位置     │
│                      │  モンスター数 │
└──────────────────────┴──────────────┘
│              メッセージログ            │
└────────────────────────────────────┘
```

### ダンジョン全体図（別画面）
- どの種族がどのスポーナーを支配しているかを色分けで表示
- 封印済みスポーナーは別アイコンで表示
- プレイヤー・NPCの現在位置を表示
- 探索済みエリアのみ迷路構造が見える（霧のかかった探索システム）

## ゲームループ全体図

```
【ゲーム開始】
  ↓
人間族拠点に帰還・状況確認
  ├─ ダンジョン全体図で陣地状況を確認
  ├─ NPCへの指示を更新（守備/探索）
  └─ Goldで装備強化・NPC増員
  ↓
プレイヤーがダンジョンへ潜入（3D探索）
  ├─ モンスター同士の戦闘結果を確認
  ├─ 生態系の流れを読んで封印順を判断
  ├─ 守衛を倒してスポーナーを封印
  └─ Gold・素材を回収して帰還
  ↓
【繰り返し】
  ↓
全スポーナー封印 → ダンジョン解放 → CLEAR！
  または
人間族拠点が占拠される → GAME OVER
```

## プレイヤーが習得していく熟達スキル

| フェーズ | 習得する知識・判断 |
|---|---|
| 序盤 | モンスターを避けながら探索し、弱いスポーナーを封印する |
| 中盤 | モンスター同士の戦いを観察し、漁夫の利でスポーナーを封印する |
| 上級 | 生態系の流れを予測し、溢れ出しが起きる前にNPCを先回りで配置する |
| 達人 | 装備ビルドとNPC運用を最適化し、最短でダンジョン全体を解放する |

## 技術スタック

- **言語**：JavaScript（バニラ）
- **描画**：Canvas API
- **実行環境**：ブラウザ
- **3D描画**：レイキャスティング方式（既存実装）
- **迷路生成**：DFS再帰バックトラッカー（既存実装）
- **モンスターAI**：BFS経路探索ベース（既存実装・拡張予定）
