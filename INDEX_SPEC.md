# index.html（3Dダンジョン）仕様書

> 3D版の仕様・操作設計・アセット定義をまとめるドキュメント。
> `playtest.html` の2Dゲームデザインを踏襲し、一人称視点3Dで再構築する。
> 実装手順は [ROADMAP.md](ROADMAP.md) を参照。

---

## 位置づけ

```
prototype.html  →  playtest.html  →  index.html（本仕様書）
（バランス調整）    （デザイン確認）    （3D移植・完成）
```

`playtest.html` のゲームロジック（種族AI・クリスタル・バトル・ショップ・勝敗）は**ほぼそのまま流用**し、以下を3D版として独自に作り込む：

- 一人称レイキャスティング3Dビュー
- グリッドベース一人称移動（前進・後退・左右回転）
- 4方向スプライト表示（前後左右）
- ミニマップ・コンパス・霧of戦争
- UIレイアウト（3Dビュー＋ミニマップ＋ステータス＋メッセージログ）

## 操作

### 通常時（3D探索中）

| キー | 動作 | 備考 |
|---|---|---|
| W / ↑ | 前進1マス | ターン進行 |
| S / ↓ | 後退1マス | ターン進行 |
| A / ← | 左に90度回転 | ターン**非**進行 |
| D / → | 右に90度回転 | ターン**非**進行 |
| マウスホイール | 前後移動 | 補助操作 |
| Q | 自陣クリスタル上で回復 | ターン進行 |
| E | 自陣クリスタル上でショップを開く | ターン進行 |
| R | 新しい迷路（デバッグ用・後に削除） | — |

- **回転はターンを消費しない**（向きを変えるだけ）
- **前進/後退はターンを消費**（全AIが1アクション進む）
- 移動・回転アニメーション中は入力を受け付けない

### バトル中

| キー | 動作 |
|---|---|
| ↑ ↓ | ターゲット選択（上下） |
| ← → | コマンド切り替え（戦う / 逃げる） |
| Enter | コマンド実行 |

### ショップ中

| キー | 動作 |
|---|---|
| ↑ ↓ | 商品選択 |
| Enter | 購入 |
| Esc | 閉じる（ターン消費なし） |

## 世界の構造

### 迷路

- グリッドサイズ：**51 × 51 固定**
- 生成アルゴリズム：DFS再帰バックトラッカー（`generateMaze`）
- ループ追加：200カ所程度（`addLoops` で回り道を増やす）
- セル種別：0=通路、1=壁

### 4陣営のゾーン（playtest仕様を踏襲）

| 陣営 | ゾーン（行, 列） | 拠点色 |
|---|---|---|
| 人間族 | (0〜12, 0〜12) 左上 | `#4488ff` |
| ゴブリン | (0〜12, 38〜50) 右上 | `#44cc44` |
| リザード | (38〜50, 0〜12) 左下 | `#ff8844` |
| オーガ | (38〜50, 38〜50) 右下 | `#cc44cc` |

### クリスタル配置

- 各陣営ゾーンに**3個**（合計12個）
- 各陣営ゾーン隣接エリアに**中立2個**（合計8個）
- マップ中央部に**中立8個**
- 合計 **28個**

ミニマップではクリスタル所有者色のドットで表示。

## プレイヤー

### 初期ステータス

| 略称 | 名称 | 初期値 |
|---|---|---|
| HP | Hit Points | 40 |
| ATK | Attack | 7 |
| REC | Recovery | 5（自陣クリスタル上の1ターン回復量） |
| AGI | Agility | 10（逃走成功率に影響） |

### 装備スロット

- 武器 `weapon` × 1
- 防具 `armor` × 1
- アクセサリ `accessory` × 1

新規購入時、既存装備は**半値で自動買取**。装備プールは playtest.html と同一（14種）。

### 死亡時処理

- 人間クリスタルが1つだけ残っている状態で死亡 → そのクリスタルが敵に陥落 → **ゲームオーバー**
- 2つ以上残っている場合：
  - 拠点クリスタルにリスポーン
  - GOLD半減
  - 装備はそのまま維持
  - HPは最大値まで回復

## バトル仕様

### 参加ユニット

- **敵側**：接触した1体＋同種族で接触セルから隣接8マス以内（最大5体）
- **味方側**：プレイヤー＋人間族AIでプレイヤーから隣接8マス以内（最大4体、計5体）

### ターン進行

- ターン制コマンドバトル
- コマンド：戦う（ターゲット手動選択）/ 逃げる
- **世界時間はバトル中も進む**（外のAIは動き続ける）

### 逃走成功率

`min(95%, max(10%, 50% + (AGI - 10) × 2%))`

### 報酬

- モンスター撃破で **10〜20 GOLD** 獲得
- プレイヤー・味方AI両方の撃破でGOLD加算

## アセット一覧

### モンスター・NPCスプライト（24枚）

`assets/monsters/` 配下に配置。6種族 × 前後左右4方向 = 24枚。

```
assets/monsters/
├── human_front.png      human_back.png      human_left.png      human_right.png
├── elf_front.png        elf_back.png        elf_left.png        elf_right.png
├── dwarf_front.png      dwarf_back.png      dwarf_left.png      dwarf_right.png
├── goblin_front.png     goblin_back.png     goblin_left.png     goblin_right.png
├── lizard_front.png     lizard_back.png     lizard_left.png     lizard_right.png
└── ogre_front.png       ogre_back.png       ogre_left.png       ogre_right.png
```

**画像仕様（推奨）**
- フォーマット：PNG（透過あり）
- サイズ：縦長推奨（例：128 × 192 px）、全ファイル**同じアスペクト比**
- 背景：完全透過
- 影：キャラクター足元に落とす（地面と馴染ませる）
- 視点：プレイヤーとほぼ同じ目線高さ（地面から等身大）

**方向の意味**
ビルボードレンダリングで、モンスターの `facing`（向き）とプレイヤーから見た角度から選択：

```
                  モンスター正面方向
                       ↑
              ┌───────────────────┐
              │    front.png       │
              │  （プレイヤーと    │
              │   向かい合う）     │
              ├─────┬─────┬─────┤
              │left │     │right│
              │.png │     │.png │
              ├─────┴─────┴─────┤
              │    back.png        │
              │  （プレイヤーに    │
              │   背を向ける）     │
              └───────────────────┘
```

## 技術仕様

### 技術スタック

- 言語：JavaScript（バニラ、ESモジュール不使用）
- 描画：Canvas API
- 実行環境：ブラウザ（ローカルファイル直起動も可）
- 依存ライブラリ：なし

### ファイル構成（js/配下）

`<script>` は index.html から**以下の順**で読み込む（依存順）：

```
1.  js/config.js       定数
2.  js/maze.js         迷路・BFS
3.  js/player.js       プレイヤー状態・移動
4.  js/monsters.js     ユニットAI
5.  js/crystals.js     クリスタル管理
6.  js/render3d.js     レイキャスト描画
7.  js/minimap.js      ミニマップ描画
8.  js/battle.js       バトルロジック
9.  js/shop.js         ショップロジック
10. js/ui.js           サイドバー・メッセージログ・リザルト
11. js/input.js        キー入力
12. js/main.js         ゲームループ・init
```

### レンダリング定数（config.js）

```js
CANVAS_W = 1000
CANVAS_H = 700
GRID_SIZE = 51
CELL_SIZE = 40
RAY_COUNT = 300
WALL_HEIGHT_CONST = 18000
MOVE_FRAMES = 8           // 1マス移動フレーム数
ROT_FRAMES  = 6           // 90度回転フレーム数
MINIMAP_VIEW_CELLS = 7    // ミニマップ表示範囲
```

### モンスターAI階層（Phase 5以降）

```
┌─────────────────────────────────────┐

                   ↓

│  - 目標クリスタル選択                  │
```

## playtest.html との差分まとめ

| 項目 | playtest.html | index.html（本仕様） |
|---|---|---|
| 視点 | 2D俯瞰 | 一人称3D |

| マップ表示 | 26×26セル追従 | ミニマップ7×7追従 |
| 霧 | なし | なし（探索済みマーキングは残す） |
| UI | サイドバー（HTML） | サイドバー（一部Canvas描画） |
| バトルUI | HTMLモーダル | HTMLモーダル（流用） |
| ショップUI | HTMLモーダル | HTMLモーダル（流用） |
