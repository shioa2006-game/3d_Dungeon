# index.html（3Dダンジョン）仕様書

> 3D版の仕様・操作設計・アセット定義をまとめるドキュメント。
> `playtest.html` の2Dゲームデザインを踏襲し、一人称視点3Dで再構築する。
> 実装手順は [ROADMAP.md](ROADMAP.md) を参照。

---

## 位置づけ

```
prototype.html  →  playtest.html  →  index.html（本仕様書）
（バランス調整）    （デザイン確認）    （3D移植・完成）
```

`playtest.html` のゲームロジック（種族AI・クリスタル・バトル・ショップ・勝敗）は**ほぼそのまま流用**し、以下を3D版として独自に作り込む：

- 一人称レイキャスティング3Dビュー
- グリッドベース一人称移動（前進・後退・左右回転・左右ストレーフ）
- 4方向スプライト表示（前後左右）
- ミニマップ・コンパス・全体マップ（Mキー）
- UIレイアウト（3Dビュー＋ミニマップ＋Message Log＋Status＋Factions）

## 操作

### 通常時（3D探索中）

| キー | 動作 | 備考 |
|---|---|---|
| W / ↑ | 前進1マス | ターン進行 |
| S / ↓ | 後退1マス | ターン進行 |
| A / ← | 左に90度回転 | ターン**非**進行 |
| D / → | 右に90度回転 | ターン**非**進行 |
| Q（クリスタル外） | 左ストレーフ1マス | ターン進行 |
| E（クリスタル外） | 右ストレーフ1マス | ターン進行 |
| Q（自陣クリスタル上） | 回復 | ターン進行 |
| E（自陣クリスタル上） | ショップを開く | ターン進行 |
| マウスホイール | 前後移動 | 補助操作 |
| M | 全体マップ開閉 | ターン非進行 |
| Esc | 全体マップを閉じる | ターン非進行 |
| R | 新規迷路（リスタート） | — |

- **回転はターンを消費しない**（向きを変えるだけ）
- **前進/後退/ストレーフはターンを消費**（全AIが1アクション進む）
- 移動・回転アニメーション中は入力を受け付けない
- **回転はスプリングアニメーション**（`ROT_SPRING_K` で硬さ調整）。ゲームロジック（facing）は即時確定し、描画専用の `visualAngle` が追従する

### バトル中

| キー | 動作 |
|---|---|
| ↑ ↓ | ターゲット選択（上下） |
| ← → | コマンド切り替え（戦う / 逃げる） |
| Enter / Space | コマンド実行 |

### ショップ中

| キー | 動作 |
|---|---|
| ↑ ↓ | 商品選択 |
| Enter | 購入 |
| Esc | 閉じる（ターン消費なし） |

## 世界の構造

### 迷路

- グリッドサイズ：**51 × 51 固定**
- 生成アルゴリズム：DFS再帰バックトラッカー（`generateMaze`）
- ループ追加：200カ所程度（`addLoops` で回り道を増やす）
- セル種別：0=通路、1=壁
- ブロック内に通路セルがゼロの場合は迷路を再生成する（`generateMazeUntilValid`）

### 5×5ブロック分割

マップ全体をプレイ範囲（row/col 1〜49）で **5×5 = 25ブロック** に分割し、各ブロックに**クリスタル1個**を配置（計25個）。

**ブロック列幅 / 行高：10, 10, 9, 10, 10**（歪みを中央に集約）

| ブロック列index | col 範囲 |
|---|---|
| 0 | 1〜10 |
| 1 | 11〜20 |
| 2 | 21〜29 |
| 3 | 30〜39 |
| 4 | 40〜49 |

（行も同一）

### 4陣営の初期陣地（L字3ブロック）

| 陣営 | 初期ブロック（行index, 列index） | 形状 |
|---|---|---|
| 人間族（左上） | (0,0), (0,1), (1,0) | L字 |
| ゴブリン（右上） | (0,4), (0,3), (1,4) | 逆L字 |
| リザード（左下） | (4,0), (4,1), (3,0) | L字 |
| オーガ（右下） | (4,4), (4,3), (3,4) | 逆L字 |

残り**13ブロック**は中立スタート。

### クリスタル連結ルール

- 各ブロックを頂点、上下左右4方向隣接ブロックをエッジとするグラフ上で BFS を実行
- 自陣営の初期3ブロックのいずれかから連結して到達できるクリスタルを **「有効（valid）」** とする
- クリスタル所有者が変わるたびに全陣営の連結判定を再計算
- **有効クリスタルのみ** 以下の機能を利用可能：
  - スポーン（AIユニット生成）
  - 回復（`healAtCrystal`）
  - ショップ（`openShop`）
- 飛び地クリスタルの見た目は通常の陣営色のまま（グレーアウトなし）
- プレイヤーが飛び地クリスタル上で Q/E を押してもターン消費せずログ警告のみ
- ユニット上限の計算は**所有クリスタル全体**（連結問わず）の合計で行う

ミニマップではブロック占有陣営の色が**占領に連動してリアルタイムに変化**する。

### スポーン間隔（ターン数）

| 陣営 | 間隔 |
|---|---|
| 人間族 | 10（human/elf/dwarf順にローテーション） |
| ゴブリン | 11 |
| リザード | 13 |

### ユニット上限

- 陣営の所有クリスタル数（連結問わず全体） × 5（最大100）
- プレイヤーは**別枠**

### NPCアグロ

モンスター陣営のユニットはプレイヤーが `aggroRange` 以内に入ると追跡を開始し、隣接1マスまで到達したときにバンプ攻撃でターン制バトルを起動する。retreating / healing 中はアグロ無効。

| 陣営 | aggroRange | 意図 |
|---|---|---|
| ゴブリン | 1 | 狡猾・慎重。隣接マスのみ反応 |
| リザード | 2 | バランス型。2マス以内で追跡 |
| オーガ | 3 | 好戦的。3マス以内で積極追跡 |

### 相性補正（AFFINITY_DEBUFF）

攻撃側のATKが×0.7になる組み合わせ：

| 攻撃 | 防御 | 補正 |
|---|---|---|
| human → goblin | | ×0.7 |
| elf → lizard | | ×0.7 |
| dwarf → ogre | | ×0.7 |
| goblin → ogre | | ×0.7 |
| lizard → goblin | | ×0.7 |
| ogre → lizard | | ×0.7 |

## プレイヤー

### 初期ステータス

| 略称 | 名称 | 初期値 |
|---|---|---|
| HP | Hit Points | 40 |
| ATK | Attack | 7 |
| REC | Recovery | 5（自陣クリスタル上の1ターン回復量） |
| AGI | Agility | 10（逃走成功率に影響） |

### 装備スロット

- 武器 `weapon` × 1
- 防具 `armor` × 1
- アクセサリ `accessory` × 1

新規購入時、既存装備は**半値で自動買取**。装備プールは playtest.html と同一（14種）。

### 死亡時処理

- 人間クリスタルが1つだけ残っている状態で死亡 → そのクリスタルが敵に陥落 → **ゲームオーバー**
- 2つ以上残っている場合：
  - 有効な自陣クリスタルにリスポーン（有効クリスタルが無い場合は全所有クリスタルからフォールバック）
  - GOLD半減
  - 装備はそのまま維持
  - HPは最大値まで回復

## バトル仕様

### バトル開始トリガー

1. **プレイヤーバンプ**：移動先マスに敵がいた場合、通過せずその場でバトル起動（`startMove` 内で検出）
2. **NPCアグロバンプ**：アグロ状態の敵がターン移動後に隣接1マスに入ったとき起動（`checkMonsterBumpPlayer` で検出）

### 参加ユニット

- **敵側**：接触した1体＋同種族で接触セルから隣接8マス以内（最大5体）
- **味方側**：プレイヤー＋人間族AIでプレイヤーから隣接8マス以内（最大4体、計5体）

### ターン進行

- ターン制コマンドバトル
- コマンド：戦う（ターゲット手動選択）/ 逃げる
- **世界時間はバトル中も進む**（外のAIは動き続ける）
- バトルUIは3Dビュー領域に `position: absolute` でオーバーレイ（HTML/CSS）

### 逃走成功率

`min(95%, max(10%, 50% + (AGI - 10) × 2%))`

### 報酬

- モンスター撃破で **10〜20 GOLD** 獲得
- プレイヤー・味方AI両方の撃破でGOLD加算

## アセット一覧

### モンスター・NPCスプライト（24枚）

`assets/monsters/` 配下に配置。6種族 × 前後左右4方向 = 24枚。

```
assets/monsters/
├── human_front.png      human_back.png      human_left.png      human_right.png
├── elf_front.png        elf_back.png        elf_left.png        elf_right.png
├── dwarf_front.png      dwarf_back.png      dwarf_left.png      dwarf_right.png
├── goblin_front.png     goblin_back.png     goblin_left.png     goblin_right.png
├── lizard_front.png     lizard_back.png     lizard_left.png     lizard_right.png
└── ogre_front.png       ogre_back.png       ogre_left.png       ogre_right.png
```

**画像仕様（推奨）**
- フォーマット：PNG（透過あり）
- サイズ：縦長推奨（例：128 × 192 px）、全ファイル**同じアスペクト比**
- 背景：完全透過
- 影：キャラクター足元に落とす（地面と馴染ませる）
- 視点：プレイヤーとほぼ同じ目線高さ（地面から等身大）

**方向の意味**
ビルボードレンダリングで、モンスターの `facing`（向き）とプレイヤーから見た角度から選択：

```
                  モンスター正面方向
                       ↑
              ┌───────────────────┐
              │    front.png       │
              │  （プレイヤーと    │
              │   向かい合う）     │
              ├─────┬─────┬─────┤
              │left │     │right│
              │.png │     │.png │
              ├─────┴─────┴─────┤
              │    back.png        │
              │  （プレイヤーに    │
              │   背を向ける）     │
              └───────────────────┘
```

## 技術仕様

### 技術スタック

- 言語：JavaScript（バニラ、ESモジュール不使用）
- 描画：Canvas API
- 実行環境：ブラウザ（ローカルファイル直起動も可）
- 依存ライブラリ：なし

### ファイル構成（js/配下）

`<script>` は index.html から**以下の順**で読み込む（依存順）：

```
1.  js/config.js       定数・数学ヘルパー・ユニット定義・相性補正・スプライト画像ロード
2.  js/sprites.js      ドット絵スプライト定義（32×32 インデックス配列＋パレット）・drawSpriteAt()
3.  js/maze.js         迷路生成・BFS経路探索
4.  js/player.js       プレイヤー状態・移動・回復・装備
5.  js/monsters.js     ユニットAI（領土争奪・アグロ・バトル接触）
6.  js/crystals.js     クリスタル管理・連結判定・スポーン
7.  js/render3d.js     レイキャスト描画・スプライト描画
8.  js/minimap.js      ミニマップ描画・全体マップ（Mキー）
9.  js/battle.js       バトルロジック・勝敗判定・死亡処理
10. js/shop.js         ショップロジック・回復処理・onCrystal管理
11. js/ui.js           Status/Factions/Message Log 描画
12. js/input.js        キー入力・ホイール入力
13. js/main.js         ゲームループ・init・newMaze
```
